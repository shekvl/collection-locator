K-Anonymizer

The K-Anonymizer is a part of the Collection Locator toolset aimed at anonymizing sensitive data prior to transferring it to the Collection Locator instance, which is able to perform search over this data.

Deployment

Every biobank which supplies the data to the Collection Locator instance has to install this component locally. It can reside behind a biobank corporate firewall, as it is not contacted by Collection Locator directly. It should, however, be able to contact Collection Locator itself.

Functionality

Below are the steps for preparing the anonymized data by means of K-Anonymizer.
  1. Local biobank administrators upload available sensitive collection data to K-Anonymizer. The collection attributes have to be annotated by OMOP CDM concepts.
  2. K-Anonymizer receives the anonymization definitions from the running Collection Locator instance, each such definition specifies a set of sensitive attributes to be anonymized, annotated with OMOP CDM concepts. This can be currently done by request of the biobank administrators or by manual file upload.
  3. Local biobank administrators select the attributes of the uploaded collection data based on the available definitions, and ask K-Anonymizer to anonymize this data. it is also possible to anonymize the whole collections.
  4. K-Anonymizer performs anonymization of the data belonging to these attributes or the whole collections by means of an anonymization algorithm (currently, a version of k-anonymity algorithm is supported) to form anonymized datasets.
  5. Finally, local biobank administrators ask K-Anonymizer to send the anonymized datasets back to the Collection Locator server, which is able to perform the search within this data.
